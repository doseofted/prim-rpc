Let's learn how to set up Prim+RPC in your own project. In this guide, we'll choose Prim+RPC plugins for your favorite
server and client frameworks. There are two categories of plugins: **Method** Plugins and **Callback** Plugins. Method
plugins handle function calls and callback plugins add support for callbacks on those functions. You can use callback
plugins alongside method plugins: Prim+RPC knows when to use which plugin.

Both types of plugins have versions intended for the server and for the client. **Plugins** are used on the client and
**Handlers** are plugins used on the server. In other words, a method plugin on the client has a corresponding method
handler (plugin) on the server, same with callbacks.

This guide is interactive to help you pick the right plugins. Let's get started!

## Installation

The first step is to install Prim+RPC. You may find instructions for your package manager below.

<CodeFile tabs={["npm", "yarn", "pnpm", "bun"]}>

```zsh npm
npm add @doseofted/prim-rpc @doseofted/prim-rpc-plugins @doseofted/prim-rpc-tooling
```

```zsh yarn
yarn add @doseofted/prim-rpc @doseofted/prim-rpc-plugins @doseofted/prim-rpc-tooling
```

```zsh pnpm
pnpm add @doseofted/prim-rpc @doseofted/prim-rpc-plugins @doseofted/prim-rpc-tooling
```

```zsh bun
bun add @doseofted/prim-rpc @doseofted/prim-rpc-plugins @doseofted/prim-rpc-tooling
```

</CodeFile>
