Let's learn how to set up Prim+RPC in your own project. In this guide, we'll choose Prim+RPC plugins for your favorite
server and client frameworks. There are two categories of plugins: **method** plugins and **callback** plugins.

Method plugins are simple: they handle a function call and send a single result. **Method plugins are required** as most
of Prim+RPC's functionality relies on them.

Callback plugins are optional and have the added ability handle callbacks given on a function, of which may fire
multiple times. **Callback plugins are optional** and are only needed if you use callbacks on your functions. You can
use callback plugins alongside method plugins: Prim+RPC knows when to use which plugin.

Both types of plugins have versions intended for the server and for the client. Plugins used on the server are referred
to as "handlers" rather than plugins to avoid confusion: a method **plugin on the client** has a corresponding method
**handler on the server**, same with callbacks.

This guide is interactive to help you pick the right plugins. Let's get started!
