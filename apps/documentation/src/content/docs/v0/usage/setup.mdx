Let's learn how to set up Prim+RPC in your own project. If you'd like to get set up quickly or are new to Prim+RPC, you
may want to check out the [Examples](/docs/v0/usage/examples).

In this guide, we'll choose Prim+RPC plugins for your favorite server and client frameworks. There are two categories of
plugins: **Method** Plugins and **Callback** Plugins. Method plugins handle function calls and callback plugins are
optional and add support for callbacks on those functions. You can use callback plugins alongside method plugins:
Prim+RPC knows when to use which plugin.

Both types of plugins have versions intended for the server and for the client. **Plugins** are used on the client and
**Handlers** are plugins used on the server. In other words, a method plugin on the client has a corresponding method
handler (plugin) on the server, same with callbacks.

This guide is interactive to help you pick the right plugins. Let's get started!

## Installation

The first step is to install Prim+RPC. The core of Prim+RPC (`@doseofted/prim-rpc{:txt}`) is framework-agnostic and
includes both server and client code. It is made compatible with frameworks by installing Prim+RPC plugins
(`@doseofted/prim-rpc-plugins{:txt}`). Prim+RPC also includes optional tooling (`@doseofted/prim-rpc-tooling{:txt}`)
that includes features like documentation generation and other helpers.

You may find instructions for your package manager below.

<CodeTabs tabs={["npm", "yarn", "pnpm", "bun"]}>

<details open>
<summary class="hidden">npm</summary>

```zsh
npm add @doseofted/prim-rpc @doseofted/prim-rpc-plugins @doseofted/prim-rpc-tooling
```

</details>

```zsh yarn
yarn add @doseofted/prim-rpc @doseofted/prim-rpc-plugins @doseofted/prim-rpc-tooling
```

```zsh pnpm
pnpm add @doseofted/prim-rpc @doseofted/prim-rpc-plugins @doseofted/prim-rpc-tooling
```

```zsh bun
bun add @doseofted/prim-rpc @doseofted/prim-rpc-plugins @doseofted/prim-rpc-tooling
```

</CodeTabs>

You may also use Prim+RPC with Deno or from a CDN.
